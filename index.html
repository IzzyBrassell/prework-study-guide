HTML

  • The Head element contains information about the webpage.
  • meta elements describes the webpage.
  • title will show up in the tab
  • use the link command to add css
  • opening the sourcecode to this studyguide will honestly help you more than any notes I could give.
  • The body element represents the visible content shown to the user.

CSS

  • start by describing the html feature that your code will style
  • width and height can be described with a percentage or a pixel ammount
  • you can align text and set colors with intuitive commands
  • use class names to differentiate html elements that use the same name like we did with class=card
  • try to avoid using id's because using p and ul will make all parts using that element follow the rules in your css.

Git

  • ls: lists the directory and files you are currently in
  • cd "insert directory": changes the current directory you are in, in addition you can return to previous directories by adding .. instead of a directory name
  • git status: checks what branch we are currently on
  • git checkout -b branch-name: creates a new branch and switches to it
  • git add -A: adds changes in current working branch to the staging area
  • git pull origin main: Pulls the main branch from the repository
  • git push origin "insert branch name": pushes the changes to github
  • Git command quick list

Javascript

  • A variable is a named container that allows us to store data in our code.
  • Control flow is the order in which a computer executes code in a script.
  • write javascript like we wrote the module one prework with the plates
  • remember to define something before writing a function with it. Javascript is executed top down
